1.1.Rx programing

위에서 Rx의 소개문구를 보았는데 여기서 주목해야할점은 비동기 프로그래밍과observable streams 이다.

  • 비동기(Async)프로그래밍은 동시성(Concurrency)프로그래밍과 다르게 프로그래밍의 흐름을 멈춰 기다리는 동작 없이 바로 다음 작업을 실행할 수 있게 하는 방식이다.

  • observable streams은 observable과 streams로 분리되는데 Reactive Programing은 기본적으로 모든 것을 스트림(stream)으로 본다. 액션, api call, 등 모든 데이터의 흐름을 시간 순서에 따라 전달되는 스트림으로 처리를 한다. 옵저버블(observable)은 observer의 메서드를 호출하면서 정보등을 호출하는 Rx의 가장 중요한 스트림의 시작 역할을 한다.

핵심은 모든것은 스트림(Stream)이고, 데이터 스트림을 옵저버블(Observable)이라 표현한다
